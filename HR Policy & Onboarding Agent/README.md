# HR Policy & Onboarding Agent

> RAG-powered HR assistant with personalised onboarding checklists, policy Q&A with citations, and a deterministic benefits calculator. The internal workflow automation equivalent of Microsoft Copilot Studio — built in pure Python.

---

## What it does

**KnowledgeBaseAgent** — the RAG engine. Ingests any HR document (PDF, DOCX, TXT, Markdown) into a local vector store, chunks it with sentence-aware overlapping windows, embeds with `sentence-transformers`, and retrieves the top-K most similar chunks for any employee question. Answers are generated by Claude using only the retrieved policy chunks as context — no hallucination, full citations.

**HRPolicyAgent** — the conversational layer on top of the RAG engine. Adds intent classification (leave, benefits, compensation, conduct, performance, onboarding, offboarding), multi-turn conversation memory, sensitive topic detection (harassment, legal, medical queries route immediately to a human HR partner), and low-confidence escalation routing. Every response includes follow-up prompt suggestions to help employees discover related policies.

**OnboardingAgent** — generates personalised onboarding checklists from a base task library plus role-specific task sets (engineer, manager, sales, finance). Computes exact due dates from the employee's start date. Tracks task status (pending → in_progress → completed | blocked) and surfaces blocked/overdue tasks to the onboarding coordinator dashboard. Plans persist to disk and survive process restarts.

**BenefitsAgent** — deterministic calculation engine for PTO balances (with part-time pro-rating and forfeiture warnings), 401(k) match calculations (tiered match formula, max-match optimisation tips), health plan cost breakdowns (all tiers, all plans), and full benefits summaries. No LLM inference — pure arithmetic from the configurable benefits catalogue JSON.

## Architecture

```
Employee (Teams / Slack / Web / API)
                │
                ▼
         FastAPI HR API (port 8002)
                │
    ┌───────────┼────────────────────┐
    │           │                    │
    ▼           ▼                    ▼
HRPolicyAgent  OnboardingAgent  BenefitsAgent
    │
    ▼
KnowledgeBaseAgent (RAG)
  ├── DocumentLoader       ← .txt / .md / .pdf / .docx
  ├── TextChunker          ← sentence-aware, overlapping windows
  ├── EmbeddingEngine      ← sentence-transformers / TF-IDF fallback
  ├── VectorStore          ← cosine similarity search (ChromaDB interface)
  └── Anthropic API        ← grounded answer generation with citations
```

## Quick start

```bash
pip install -r requirements.txt

# 1. Generate synthetic HR documents and ingest into knowledge base
python main.py --ingest

# 2. Demo: HR policy Q&A
python main.py              # runs all demos

# 3. Demo: Onboarding plan creation
python main.py --onboard

# 4. Demo: Benefits calculator
python main.py --benefits

# 5. Start REST API (port 8002)
python main.py --serve

# Run tests
pytest tests/ -v
```

## API

```bash
# Ask an HR question
curl -X POST http://localhost:8002/policy/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "How many PTO days do I get?", "employee_id": "emp-001"}'

# Create an onboarding plan
curl -X POST http://localhost:8002/onboarding/plans \
  -H "Content-Type: application/json" \
  -d '{"employee_id": "emp-002", "employee_name": "Jane Doe", "role": "Software Engineer", "department": "Engineering", "start_date": "2026-04-01"}'

# Get PTO balance
curl "http://localhost:8002/benefits/emp-001/pto?hire_date=2026-01-01&pto_used=5"

# 401(k) match calculator
curl "http://localhost:8002/benefits/retirement-match?contribution_pct=6&annual_salary=100000"

# Coordinator dashboard
curl http://localhost:8002/onboarding/coordinator/dashboard
```
